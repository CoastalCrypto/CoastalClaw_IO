## What We're Building

**CoastalClaw** is a self-hosted AI Agent OS — a private command center for AI: chat with intelligent specialist agents, schedule them on cron, build custom tools, push results to Telegram/Discord/Slack, all through a clean web UI. It is an open-source product for anyone to deploy on their own hardware.

**Repo:** `https://github.com/CoastalCrypto/CoastalClaw_IO.git` (branch: `master`)

**Stack:** Node 22 ESM, TypeScript, Fastify, WebSocket, better-sqlite3, Ollama/vLLM/AirLLM (local LLMs), React + Tailwind v4, pnpm monorepo.

```
packages/
  core/       Fastify API server, agentic loop, tool registry, model routing, SQLite persistence
  web/        React + Vite frontend
  daemon/     MCP server / background daemon
  shell/      CLI shell
  architect/  UI builder
```

- **Component:** `packages/web/src/components/ChatPane.tsx`
- **Hook:** `packages/web/src/hooks/useIsMobile.ts`
- Layout picker button in the Chat page header toolbar (grid icon)
- Options: 1, 2, 3, 4, 6, 8, 9 panes shown as SVG grid previews

- Single pane (1) shows the full existing Chat UI unchanged

### 3. OpenAI-Compatible Unified Model Client
- **File:** `packages/core/src/models/openai-compat.ts`
- `OpenAICompatibleClient` base class: chat, SSE streaming, tool use, optional `apiKey`
- `VllmClient` and `AirLLMClient` now extend it (eliminated ~80 lines of duplication)
- `InferenceClient` interface exported from `ModelRouter`
- `ModelRouter.createCompatClient(url, apiKey?)` factory for ad-hoc endpoints

### 4. Per-Agent Credential Store
- **DB:** `agent_credentials` table in `agents.db` (`agent_id`, `key`, `value`, `updated_at`)
- **Registry methods:** `getCredentials(agentId)`, `setCredential(id, key, val)`, `deleteCredential(id, key)`
- **Routes:** `GET /api/admin/agents/:id/credentials`, `PUT /api/admin/agents/:id/credentials`
- **UI:** `[CREDS]` button on each AgentCard opens a modal with key/value editor (password-masked values)

### 5. Deterministic Routing Bindings
- **DB:** `bindings TEXT` column on `agents` table (JSON array)
- **Type:** `AgentBinding { sessionPattern?: string, source?: string, priority: number }`
- **Registry method:** `findBindingMatch(sessionId, source)` — regex/source matching, highest priority wins
- **Routing:** `stream.ts` checks bindings *before* explicit `agentId` and domain routing (skips LLM classifier when matched)
- **UI:** `[BIND]` button on each AgentCard opens a modal binding rule editor

### 6. Multi-Agent Pipeline
- **File:** `packages/core/src/agents/pipeline.ts` — `AgentPipeline` class, chains stages sequentially
- **Route:** `POST /api/pipeline/run` (body: `{ stages: [{agentId}], input, sessionId? }`)
- **Page:** `packages/web/src/pages/Pipeline.tsx` — stage builder, input textarea, results accordion
- **Nav:** "Pipeline" item in admin NavBar (⛓ icon)

### 7. Mobile-Responsive Chat
- `useIsMobile` hook (matchMedia 768px)
- Left agent rail hidden on mobile
- Floating `✳` toggle button + bottom drawer for agent selection on mobile
- Input `font-size: 16px` to prevent iOS keyboard auto-zoom

## SSE Streaming Protocol (`POST /api/chat/stream`)

```
Request body: { message, sessionId?, images?, agentId? }
Headers:      x-source (optional, for binding rules)

SSE events:
  event: domain   → { domain: string }
  event: token    → { token: string }       (streaming tokens, build up reply)
  event: reply    → { reply, domain }       (non-streamed fallback — tool-use path)
  event: approval → { approvalId, agentName, toolName, cmd }
  event: error    → { message: string }
  event: done     → { sessionId, domain }
```

Agent routing priority: binding rules → explicit `agentId` → domain classifier → `general` fallback.
