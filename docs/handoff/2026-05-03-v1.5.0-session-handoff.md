## v1.5.0 Architecture — Fully Wired

The architect daemon is now a functional end-to-end system. When started (non-legacy mode), it:

1. **Polls pending work items** from `architect.db`
2. **Plans** via `ArchitectModelRouterClient` → Ollama/vLLM/AirLLM
3. **Builds** by applying diff, running lint/typecheck/build/test gates scoped to touched packages
4. **Creates PRs** via `gh pr create` with generated body + trailers
5. **Polls PR status** — transitions to merged/vetoed on close
6. **Auto-merges** PRs when `approval_policy=none`
7. **Captures Time-Travel snapshots** before builds (shadow-git repo)
8. **Emits events** to `architect_events` table (Plan 3B/C SSE seam)
9. **Runs curriculum scans** when idle + enabled (proposes improvements from codebase signals)
10. **Suppresses** rejected curriculum proposals for 30 days

### REST API surface (all admin-gated)

| Endpoint | Purpose |
|----------|---------|
| `GET/POST /api/admin/architect/work-items` | CRUD work items |
| `PATCH /api/admin/architect/work-items/:id` | Pause/resume/priority |
| `GET /api/admin/architect/activity` | Cycle timeline with filters |
| `GET /api/admin/architect/cycles/:id` | Full cycle detail |

## Cost optimization used

| Model | Used for | Savings |
|-------|----------|---------|
| Haiku | Mechanical chunks (1-2 files, clear spec) | ~70% cheaper than opus |
| Sonnet | Integration chunks (multi-file, judgment) | ~50% cheaper than opus |
| Opus | Orchestration, architecture decisions only | Full price, minimal use |
| Combined reviews | Single spec+quality pass per chunk | ~50% fewer review tokens |
| Background reviews | Review runs while next chunk implements | Wall-clock time savings |
