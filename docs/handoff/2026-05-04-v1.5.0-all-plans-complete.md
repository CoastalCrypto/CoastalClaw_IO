## What was built

### Daemon (`packages/architect/src/`)
- **3-phase tick loop**: poll active PRs, process pending work items, idle-time curriculum scan
- **Stage pipeline**: planning -> building (4 gates) -> PR creation -> PR review (polling)
- **Time-Travel snapshots**: shadow-git repo at `data/.architect-snapshots/`
- **Curriculum scanner**: harvests codebase signals (TODOs, churn), proposes improvements via LLM
- **Model router adapter**: bridges real ModelRouter to architect's interface
- **Event log**: `architect_events` table for SSE streaming
- **Notifications**: HMAC-signed callback tokens, ArchitectNotifier for Output Channels
- **CLI**: `coastal-ai architect on|off|status|mode|ask|queue|approve|reject|last|digest`

### REST API (`packages/core/src/api/routes/architect*.ts`)
- Work-item CRUD with status filters + PATCH
- Cycle activity timeline with stage/time filters
- Approval endpoint (approve/revise/reject)
- Power on/off + mode switch controls
- Insights aggregate stats
- Receipts (merged PRs attribution)
- HMAC callback endpoint for notification buttons
- SSE event stream

### Web UI (`packages/web/src/pages/Architect.tsx`)
- **Status card**: "What's happening right now" with power indicator
- **Tab navigation**: Missions | Activity | Insights | Receipts | Settings
- **Missions (Queue)**: work item list, inline create form, pause/resume/cancel actions
- **Activity**: cycle timeline with inline expansion, filter chips, plan/test/PR detail
- **Approval flow**: approve/revise/reject buttons with comment input
- **Insights**: 6 stat tiles (success rate, iterations, time saved, queue, errors, top failure)
- **Receipts**: merged PR list with attribution
- **Settings**: power toggle, mode selector (3 tiles), Run Now button
